# Review Command

## Description

Review code changes, plans, or documentation and provide concise, actionable recommendations for improvement.

## Trigger

User asks for a review of code, plan, or documentation.

## Steps

1. **Assess**: Evaluate clarity, structure, and consistency.
2. **Identify**: Spot potential issues, weaknesses, or risks.
3. **Suggest**: Propose optimizations or best practices.
4. **Align**: Ensure compliance with existing project standards (`cursor-stuff/.cursor/rules/coding.mdc`).

## Output Format

- **Summary:** Brief overview of current quality.
- **Issues:** List of detected problems or risks.
- **Recommendations:** Specific improvements or next steps.
