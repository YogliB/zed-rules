# Review Command

## Purpose

Review code changes, plans, or documentation and provide concise, actionable recommendations for improvement.

## Behavior

- Assess clarity, structure, and consistency.
- Identify potential issues or weaknesses.
- Suggest optimizations or best practices.
- Focus on maintainability, readability, and alignment with existing project standards (`cursor-stuff/.cursor/rules/coding.mdc` file)

## Output Format

- **Summary:** Brief overview of current quality.
- **Issues:** List of detected problems or risks.
- **Recommendations:** Specific improvements or next steps.
